## 1. カメラ映像の生成AI入力化（視覚をもつ谺）

**状態: サスペンド（2026-07-07）**

在室検知用のIPカメラ映像を，Claude への視覚入力として活用し，「部屋の様子を見ながら答える」谺を実現する．

### 実現方針

生成AI（Claude含む）は動画を直接受け取れないため，**フレームサンプリング方式**を採る．既存のRTSP接続と `spawnMjpegStream`（在室検知・ライブプレビュー用）でフレーム切り出しの足回りは揃っており，JPEGフレームを Claude API の image ブロックとして渡すだけで最小構成が動く．

段階的な実装案:

1. **最小の第一歩**: 発話処理時に最新フレームを1枚添付し，「見ながら答える」応答を実現する（既存コードの延長で実装可能）
2. **イベント駆動の観察**: 在室状態の変化や発話があった瞬間だけフレームを送る（コスト最適化）
3. **時間方向の理解**: 複数フレームをまとめて渡す，またはローカルで差分要約してから渡すことで「何が起きたか」を追う

### 設計上の論点

- **コスト**: 1フレームは概ね1,000〜1,600トークン程度．毎秒送る連続理解は非現実的なので，「動き検出はローカル（既存のONNX人物検出），意味理解が必要な瞬間だけクラウド」という在室判定ハイブリッド化（§12系）と同じ設計思想を貫く
- **プライバシー**: 自室映像を外部APIへ送ることになるため，送信条件（タイミング・トリガー）を明示的に絞る設計とし，設定画面からオン/オフできるようにする
- **レイテンシ**: 発話時添付なら応答生成と同一リクエストに載るため追加往復は発生しない
